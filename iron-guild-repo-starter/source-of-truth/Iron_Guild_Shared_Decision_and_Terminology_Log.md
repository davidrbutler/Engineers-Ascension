# Iron Guild Shared Decision and Terminology Log

## Purpose
This document preserves shared historical memory across the Iron Guild project.

It exists to record:
- naming lineage,
- model evolution,
- major cross-document decisions,
- and why important changes were made.

This file is not the main operating source for campaign, roadmap, prep-system, or quest-state rules.
It exists so those documents can stay cleaner while important decision memory is still preserved.

When in conflict, the current source-of-truth documents govern their owned areas.

---

## Ownership boundary

### This document owns
- shared terminology lineage
- cross-document decision memory
- major structural change history
- high-level rationale for important model changes

### This document does **not** own
- live campaign rules
- roadmap ordering
- prep-system execution rules
- current quest state
- detailed build instructions

## Update rule
Update this file when:

- or a cross-document change needs durable rationale.

## Model-evolution notes
- avoiding “acts” framing helped the roadmap feel like an adventure rather than a staged product reveal
- the hidden Blessing replaced the visible Gate of Readiness to reduce anxious bargaining
- meta language showed up in earlier static artifacts partly because quest structure had not been fully defined before artifact work began
- skill / pattern notes replaced the heavier “lore note” framing as the live model for pre-entry technique artifacts
- Quest Brief and Quest Log were split because “quest log” was previously doing planning and retrospective jobs at the same time
- presentation-layer development was deliberately moved out of MVP scope to prevent aesthetic work from slowing usable prep progress
